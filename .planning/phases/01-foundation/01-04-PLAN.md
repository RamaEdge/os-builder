The current Containerfile.microshift (created in Phase 1) likely uses a wildcard COPY for systemd files:
  COPY systemd/*.service /usr/lib/systemd/system/

Since observability-deploy.service is deleted in STEP 1, the wildcard COPY will no longer include it.
No change to the COPY line is needed IF it uses a wildcard. If the Containerfile explicitly names
the service files, remove the observability-deploy.service line.

a) Check the current COPY pattern:
   grep "observability-deploy\|systemd.*service\|COPY systemd" os/Containerfile.microshift

b) Remove observability-deploy.service from the systemctl enable line:
   Find the `systemctl enable` line and remove `observability-deploy.service` from it.
   The line may look like:
   ```
   systemctl enable sshd chronyd systemd-resolved podman.socket crio \
                   otelcol edge-setup.service observability-deploy.service \
                   microshift-kubeconfig-setup.service \
                   bootc-fetch-apply-updates.timer 2>/dev/null || true
   ```
   Remove "observability-deploy.service \\" or "observability-deploy.service" from this line.

c) Add the COPY line for manifests.d BEFORE the existing configuration COPY lines:
   Find the COPY section that handles configs/manifests/systemd files.
   Add after `COPY manifests/ /etc/microshift/manifests/` (or wherever manifests are copied):
   ```dockerfile
   COPY manifests/manifests.d/ /usr/lib/microshift/manifests.d/
   ```

   NOTE on benign side-effect: If the existing `COPY manifests/ /etc/microshift/manifests/` line is
   retained (as it should be), Docker/Podman will transitively copy manifests.d/ into
   /etc/microshift/manifests/manifests.d/ as a subdirectory. This is benign: MicroShift's
   kustomizePaths scans /etc/microshift/manifests.d/* (a top-level path), NOT subdirectories of
   /etc/microshift/manifests/. The files in /etc/microshift/manifests/manifests.d/ will never be
   picked up by kustomizePaths and cause no conflicts. Do NOT remove the existing manifests/ COPY
   to avoid this; simply add the new manifests.d/ COPY alongside it.
