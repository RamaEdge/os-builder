# Plan 01-03 Summary: Extend embed script + edgeworks app images

## What Was Implemented

Extended `os/scripts/embed-microshift-images.sh` with two new sections appended after the MicroShift manifest loop. Created `os/configs/edgeworks-images.txt` with 15 application image refs. Updated `os/Containerfile.microshift` to COPY the image list before the embed RUN.

**Files created/modified:**
- `os/scripts/embed-microshift-images.sh` — extended with app image processing + runtime script generation
- `os/configs/edgeworks-images.txt` — new file with 15 edgeworks application image refs
- `os/Containerfile.microshift` — added `COPY configs/edgeworks-images.txt /usr/share/edgeworks/image-list.txt`

## Key Implementation Details

### Section A: App Image Processing

Reads `/usr/share/edgeworks/image-list.txt` (populated at build time from `edgeworks-images.txt`). Processes each non-comment, non-empty line: increments TOTAL, sha256-names directories, idempotent cache check, `skopeo copy --all --preserve-digests`, appends to `image-list.txt`. Appears BEFORE runtime script generation to ensure image-list.txt is fully populated.

### Section B: Runtime Script Generation (OFFL-02)

Generates `/usr/bin/microshift-copy-images` via heredoc (COPY_EOF delimiter). Runtime script:
- Reads `/usr/lib/containers/storage/image-list.txt`
- Copies each `dir:${sha}` to `containers-storage:${img}` using `skopeo copy --preserve-digests` (NOT --all)
- Graceful: skips if no image list found

### Systemd Drop-in Generation (OFFL-03)

Creates `/usr/lib/systemd/system/microshift.service.d/microshift-copy-images.conf` with `ExecStartPre=/usr/bin/microshift-copy-images`. Runs before microshift.service on every boot.

### edgeworks-images.txt (OFFL-04)

15 images: 9 harbor.theedgeworks.ai/edgeworks/* services + 6 infrastructure (dex, external-mdns, cert-manager x3, otelcol-contrib). Comments included per plan spec.

### Containerfile Ordering

```

COPY scripts/embed-microshift-images.sh /tmp/...                         [line 59]

```
