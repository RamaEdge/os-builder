Key design references from docs/bundle-cli-design.md:

**Verify checks (design doc section 3.2):**
1. `manifest.json` exists and is valid JSON conforming to schema
2. `checksums.sha256` exists and is well-formed
3. OCI tarball file referenced in manifest exists
4. SHA256 of tarball matches `checksums.sha256` and `manifest.image.digest`
5. File size matches `manifest.image.size_bytes`
6. `schema_version` is a supported version ("1.0")

**Exit codes:** 0 = valid, 1 = verification failed, 2 = bundle not found

**Human output format (design doc):**
```
Verifying bundle: /media/usb/edgeworks-bundle-1.2.0/

  [OK] manifest.json schema valid
  [OK] checksums.sha256 well-formed
  [OK] edge-os-1.2.0.oci.tar exists (2.0 GiB)
  [OK] SHA256 checksum matches
  [OK] File size matches manifest

Bundle is valid.
  Image:   harbor.theedgeworks.ai/edgeworks/edge-os:1.2.0
  Version: 1.2.0
  Created: 2026-03-01T12:00:00Z
```

**JSON output format:** `--json` flag produces `{"status":"ok"|"failed", "checks":[...], "errors":[...]}` structure.

**Prior phase context:**
- Phase 4 creates `manifest.rs` with `BundleManifest` struct (load/save/validate helpers) and `error.rs` with `BundleError` enum.
- Phase 5 creates `create.rs` with SHA256 checksum computation using the `sha2` crate. The checksum utility can be reused or extracted.
- `main.rs` already has clap subcommand routing for `create`, `verify`, `inspect` and `--json` global flag.

  <action>
Create `crates/bundle-cli/src/verify.rs` with:

1. A `VerifyResult` struct that collects individual check results:
   ```rust
   pub struct CheckResult {
       pub name: String,
       pub passed: bool,
       pub detail: String,
   }

   pub struct VerifyResult {
       pub valid: bool,
       pub checks: Vec<CheckResult>,
       pub manifest: Option<BundleManifest>,
   }
   ```

2. A `pub fn run_verify(bundle_dir: &Path) -> Result<VerifyResult, BundleError>` function that:
   - First checks if `bundle_dir` exists. If not, return a specific error that main.rs maps to exit code 2.
   - Check 1: Load and parse `manifest.json` via `BundleManifest::load()`. Validate required fields present.
   - Check 2: Verify `schema_version == "1.0"`. Use `BundleError::UnsupportedSchema` for mismatch.
   - Check 3: Read `checksums.sha256`, validate it has the expected format (hex_hash two-spaces filename).
   - Check 4: Verify the OCI tarball file referenced in `manifest.image.file` exists on disk.
   - Check 5: Compute SHA256 of the tarball (reuse streaming hash from create.rs or implement inline using `sha2` crate). Compare against both `checksums.sha256` content and `manifest.image.digest`.
   - Check 6: Compare actual file size with `manifest.image.size_bytes`.
   - Collect all check results. If any check fails, `valid = false`.

3. A `pub fn format_verify_human(result: &VerifyResult, bundle_dir: &Path) -> String` function that formats output matching the design doc's human-readable format with [OK]/[FAIL] prefixes.

4. A `pub fn format_verify_json(result: &VerifyResult, bundle_dir: &Path) -> String` function that serializes to JSON: `{"status":"ok"|"failed", "directory":"...", "checks":[{"name":"...","passed":true,"detail":"..."}], "errors":[...]}`.

For SHA256 computation, use streaming reads with `sha2::Sha256` and `std::io::Read` to handle large files without loading into memory:
```rust
use sha2::{Sha256, Digest};
fn compute_sha256(path: &Path) -> Result<String, BundleError> {
    let mut file = std::fs::File::open(path)?;
    let mut hasher = Sha256::new();
    std::io::copy(&mut file, &mut hasher)?;
    Ok(format!("{:x}", hasher.finalize()))
}
```

For checksums.sha256 parsing, expect format: `<64-char-hex>  <filename>\n` (two spaces between hash and filename, per GNU coreutils sha256sum format).

Write comprehensive unit tests in the same file using `#[cfg(test)] mod tests`. Tests should create temporary bundle directories using `tempfile::TempDir`:
- `test_verify_valid_bundle`: Create a temp dir with a small test file, valid checksums.sha256, valid manifest.json -> verify returns valid.
- `test_verify_nonexistent_path`: Pass a path that doesn't exist -> appropriate error.
- `test_verify_corrupted_checksum`: Write wrong hash in checksums.sha256 -> ChecksumMismatch.
- `test_verify_missing_tarball`: manifest.json references a file that doesn't exist -> FileNotFound.
- `test_verify_bad_schema_version`: manifest.json has schema_version "2.0" -> UnsupportedSchema.
- `test_verify_size_mismatch`: manifest.image.size_bytes doesn't match actual file size -> SizeMismatch.
- `test_verify_missing_checksums_file`: No checksums.sha256 in bundle dir -> error.
- `test_verify_malformed_manifest`: Invalid JSON in manifest.json -> ManifestInvalid.

Add `tempfile` to dev-dependencies in Cargo.toml if not already present. Use `serde_json::to_string_pretty` to write test manifest files.
