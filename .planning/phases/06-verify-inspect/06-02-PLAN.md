Key design reference from docs/bundle-cli-design.md section 3.3:

**Inspect human output format:**
```
Bundle: /media/usb/edgeworks-bundle-1.2.0/

  Schema version: 1.0
  Created:        2026-03-01T12:00:00Z by edgeworks-bundle v0.1.0
  Image:          harbor.theedgeworks.ai/edgeworks/edge-os:1.2.0
  Version:        1.2.0
  Size:           2.0 GiB
  Digest:         sha256:abc123...
  Target device:  any
  Notes:          Hotfix for OPC-UA adapter timeout
```

**JSON output:** When `--json` is passed, output the full manifest as JSON (essentially re-serialize the loaded manifest).

**Critical constraint:** Inspect must NOT compute checksums. It only reads and displays `manifest.json`. This makes it instant even on bundles with multi-GB tarballs.

**Data types (from manifest.rs, created in Phase 4):**
```rust
pub struct BundleManifest {
    pub schema_version: String,
    pub created_at: DateTime<Utc>,
    pub created_by: String,
    pub image: BundleImage,
    pub target_device: String,
    pub notes: String,
}

pub struct BundleImage {
    pub reference: String,
    pub file: String,
    pub digest: String,
    pub size_bytes: u64,
    pub version: String,
}
```

    - Test: valid bundle directory with correct manifest.json -> displays all manifest fields in human-readable format
    - Test: valid bundle with --json flag -> outputs manifest as JSON
    - Test: bundle directory without manifest.json -> returns ManifestNotFound error
    - Test: bundle with malformed manifest.json -> returns ManifestInvalid error
    - Test: nonexistent bundle path -> returns appropriate error
    - Test: inspect does NOT read or hash the tarball file (verify by checking no file I/O on the .oci.tar)
  </behavior>
  <action>
Create `crates/bundle-cli/src/inspect.rs` with:

1. A `pub fn run_inspect(bundle_dir: &Path) -> Result<BundleManifest, BundleError>` function that:
   - Checks if `bundle_dir` exists. If not, return error (main.rs maps to exit 2).
   - Loads `manifest.json` from the bundle directory using `BundleManifest::load()`.
   - Returns the loaded manifest. That's it. No checksum computation, no file existence checks beyond the manifest itself.

2. A `pub fn format_inspect_human(manifest: &BundleManifest, bundle_dir: &Path) -> String` function that formats output matching the design doc exactly:
   ```
   Bundle: {bundle_dir}

     Schema version: {schema_version}
     Created:        {created_at} by {created_by}
     Image:          {image.reference}
     Version:        {image.version}
     Size:           {human_readable_size(image.size_bytes)}
     Digest:         {image.digest}
     Target device:  {target_device}
     Notes:          {notes}
   ```
   - Use a helper to format `size_bytes` as human-readable (e.g., "2.0 GiB"). Implement a simple `fn format_size(bytes: u64) -> String` that handles B, KiB, MiB, GiB, TiB with one decimal place.
   - If `notes` is empty, either show "—" or omit the line (prefer showing "—" for completeness).

3. A `pub fn format_inspect_json(manifest: &BundleManifest) -> String` function that serializes the manifest to pretty-printed JSON using `serde_json::to_string_pretty()`.

Write unit tests using `tempfile::TempDir`:
- `test_inspect_valid_bundle`: Create temp dir with valid manifest.json, call run_inspect -> returns BundleManifest with all fields correct.
- `test_inspect_json_output`: Call format_inspect_json on a manifest -> output is valid JSON that deserializes back to same manifest.
- `test_inspect_human_output`: Call format_inspect_human -> output contains all expected fields (schema_version, created_at, image reference, version, size, digest, target_device).
- `test_inspect_missing_manifest`: Temp dir with no manifest.json -> ManifestNotFound error.
- `test_inspect_malformed_manifest`: Write invalid JSON to manifest.json -> ManifestInvalid error.
- `test_inspect_nonexistent_path`: Path that doesn't exist -> error.
- `test_format_size`: Unit test for the size formatter: 0 -> "0 B", 1024 -> "1.0 KiB", 1073741824 -> "1.0 GiB", 2147483648 -> "2.0 GiB".

   - Call `inspect::run_inspect(&bundle_dir)`.

     - If error indicates bundle directory not found, exit with code 2.
     - If error indicates manifest issues, exit with code 1.
     - Print error message to stderr.
   - Use `std::process::exit()` for explicit exit codes, same pattern as verify.

4. Ensure `mod inspect;` is in the module declarations at the top of main.rs.
