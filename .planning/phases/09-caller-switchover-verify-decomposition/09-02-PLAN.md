Decompose run_verify() into 6 named check_* functions with a short orchestrator, switch verify.rs to shared format_bytes, and add CKSM-03 checksum filename cross-reference.

Purpose: This is the core refactoring of Phase 9. The 230-line monolithic run_verify() becomes a ~30-line coordinator over individually testable check functions. Output: Fully refactored verify.rs with 6 check_* functions, orchestrator, format import, CKSM-03 cross-reference, and 2 new tests.

**Step 7: Write the orchestrator.**
Replace the body of `run_verify()` with the orchestrator pattern. The orchestrator:
1. Checks bundle_dir exists (same guard as before)
2. Calls each check function in order
3. Short-circuits on failure (returns early with valid=false)
4. Passes data forward between checks (manifest from check_manifest, cs_line from check_checksums_file, tarball_size from check_tarball_exists)
5. Uses `manifest.clone()` for `VerifyResult.manifest` ONLY when needed — after check_manifest succeeds, `manifest` is in scope for all subsequent early returns

**Critical:** Do NOT use `manifest_opt = Some(manifest.clone())` at line 104 like the original. Instead, in the orchestrator after `check_manifest` returns `Some(manifest)`, use `manifest` directly. Clone into `Some(manifest.clone())` only in the `VerifyResult` struct literal (Rust requires this since `manifest` is still used for later checks).

The orchestrator should follow the exact pattern from 09-RESEARCH.md Pattern 3 "Orchestrator pattern" section.

**Step 8: Add CKSM-03 cross-reference.**
In the orchestrator, after `check_checksums_file` returns successfully with `cs_line`, add an inline filename cross-reference check BEFORE Check 4:
```rust
if cs_line.filename != manifest.image.file {

        ),
    });
    return Ok(VerifyResult { valid: false, checks, manifest: Some(manifest) });
}
```
This reuses the "checksums.sha256 well-formed" check name slot — on the happy path, Check 3 already pushed a passed result, and this guard only fires on mismatch. The 6-check count on the all-passing path is preserved.
