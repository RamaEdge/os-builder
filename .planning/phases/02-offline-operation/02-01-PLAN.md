Create os/scripts/embed-microshift-images.sh with EXACTLY the following content. This implements
the upstream microshift-io/microshift embed_images.sh pattern for build-time image embedding.

The script must:
1. Define IMAGE_STORAGE_DIR="/usr/lib/containers/storage", IMAGE_LIST_FILE="${IMAGE_STORAGE_DIR}/image-list.txt",
   RELEASE_DIR="/usr/share/microshift/release", COPY_SCRIPT="/usr/bin/microshift-copy-images",
   DROPIN_DIR="/usr/lib/systemd/system/microshift.service.d"
2. Validate it runs as root (id -u check)
3. Find all *.json files in RELEASE_DIR with: RELEASE_FILES=$(find "${RELEASE_DIR}" -name '*.json' -type f 2>/dev/null || true)
4. Exit gracefully if no release files found (log "No release manifests found, skipping")
5. mkdir -p IMAGE_STORAGE_DIR and truncate IMAGE_LIST_FILE with `: > "${IMAGE_LIST_FILE}"`
6. Loop over each manifest:
   - Extract images with: while IFS= read -r img; do ... done < <(jq -r '.images[]? // empty' "${manifest}" 2>/dev/null)
   - Skip empty lines
   - Increment TOTAL BEFORE any skip check: TOTAL=$((TOTAL + 1))
   - Skip registry.redhat.io images (grep -q check); in the skip branch: SKIPPED=$((SKIPPED + 1)); continue
   - Compute SHA with: SHA=$(echo -n "${img}" | sha256sum | awk '{print $1}')
   - Set DEST="${IMAGE_STORAGE_DIR}/${SHA}"
   - If DEST exists AND DEST/manifest.json exists: log "CACHED: ${img}" (no re-pull)
   - Else: log "PULL: ${img}", mkdir -p "${DEST}", run skopeo copy --all --preserve-digests "docker://${img}" "dir:${DEST}"
     - On failure: log "WARN: Failed to pull ${img}, continuing...", rm -rf "${DEST}", continue
     - On success: increment PULLED counter
   - Append "${img},${SHA}" to IMAGE_LIST_FILE
7. Log summary: "${PULLED} pulled, ${SKIPPED} skipped, ${TOTAL} total"

NOTE: Do NOT generate the runtime copy script or systemd drop-in yet — those are added in Plan 02-02.
The script in this plan handles ONLY the build-time image embedding (MicroShift system images only).
The app image list section (OFFL-04) is also Plan 02-02.

```bash
#!/bin/bash
# Embed MicroShift container images for offline/airgap operation

# Build-time: pulls MicroShift system images into /usr/lib/containers/storage as dir: blobs
# Runtime:    microshift-copy-images (generated) copies them into CRI-O's live storage
#
# Extended in Plan 02-02 to: generate microshift-copy-images script, systemd drop-in,
# and process edgeworks application images from /usr/share/edgeworks/image-list.txt
set -euo pipefail
```

Add these layers IN ORDER:

```dockerfile
# =============================================================================
# Offline container image embedding (upstream-aligned two-phase approach)
# =============================================================================
# Phase 1: Build-time — pull images into /usr/lib/containers/storage as dir: blobs
# Phase 2: Runtime — microshift-copy-images ExecStartPre copies to CRI-O storage
#
# Why two phases: /usr/lib is in the immutable bootc image layer and survives OS
# updates. CRI-O's /var storage may be wiped on updates. The ExecStartPre ensures
# images are always repopulated before MicroShift starts.
# =============================================================================

# UID/GID mapping required for skopeo during container build
RUN echo "root:1:65536" >> /etc/subuid && \
    echo "root:1:65536" >> /etc/subgid

COPY scripts/embed-microshift-images.sh /tmp/embed-microshift-images.sh

RUN chmod +x /tmp/embed-microshift-images.sh && \
    if [ "${EMBED_CONTAINER_IMAGES}" = "1" ]; then \
        /tmp/embed-microshift-images.sh; \
    fi && \
    rm -f /tmp/embed-microshift-images.sh
```
