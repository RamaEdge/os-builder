# Plan 05-02: JSON output mode and integration tests

## Goal

Add `--json` machine-readable output to the `create` command and write integration tests that verify the complete bundle creation pipeline end-to-end (including checksum verification with `sha256sum -c`).

## Context

Plan 05-01 implemented the core `create` command with human-readable output. This plan adds JSON output mode and comprehensive tests. The `--json` flag is a global flag already defined in `main.rs` by Phase 4 (via clap). This plan threads that flag through to `create::run()` and adds the JSON output path.

**Design authority:** `docs/bundle-cli-design.md` sections 3.1 (JSON output spec), 10 (acceptance criteria).

## Tasks

<task id="05-02-01">
<title>Add JSON output mode to create command</title>
<description>
Modify `create::run()` in `crates/bundle-cli/src/create.rs` to accept a `json: bool` parameter (or read it from the args struct).

When `json` is true, instead of printing the human-readable summary, output this exact JSON structure to stdout (per design doc section 3.1):

```json
{
  "status": "ok",
  "directory": "<absolute-output-dir>",
  "image": "<image-reference>",
  "version": "<version>",
  "digest": "<sha256:hex>",
  "size_bytes": <integer>,
  "files": ["manifest.json", "checksums.sha256", "<tarball-filename>"]
}
```

Implementation details:
- Use `serde_json::to_string_pretty()` to format.
- Define a `CreateOutput` struct with `#[derive(Serialize)]` for this output shape.
- When `json` is true, suppress all progress bar output. Use `indicatif::ProgressBar::hidden()` or check the json flag before creating progress bars. The JSON output must be the ONLY thing on stdout.
- On error in JSON mode, output `{"status": "error", "message": "<error description>"}` to stdout (not stderr) and exit with code 1.
- The `directory` field should be the canonicalized absolute path (use `std::fs::canonicalize()` on the output dir after creation).

Wire `--json` from `main.rs` to `create::run()`. The global `--json` flag value needs to be passed as a parameter or included in the args struct that `create::run()` receives.
</description>
</task>

<task id="05-02-02">
<title>Write integration tests for create command</title>
<description>
Create `crates/bundle-cli/tests/create_integration.rs` with integration tests.

**Test setup helper:**
- Create a function `setup_test_dir() -> tempfile::TempDir` that returns a temp directory.
- Add `tempfile = "3"` to `[dev-dependencies]` in `Cargo.toml` if not present.
- Add `assert_cmd = "2"` and `predicates = "3"` to `[dev-dependencies]` for CLI testing.

**Tests to write:**

1. **`test_create_missing_skopeo`** — Verify that when skopeo is not in PATH, the CLI prints an error containing "skopeo" and exits non-zero.
   - Set `PATH` to an empty/minimal value in the command environment to simulate missing skopeo.
   - Use `assert_cmd::Command::cargo_bin("edgeworks-bundle")` to run the CLI.
   - Assert stderr contains "skopeo" (case-insensitive).
   - Assert exit code is non-zero.

2. **`test_create_existing_output_dir`** — Verify that when output dir already contains `manifest.json`, the CLI returns an error.
   - Create a temp dir, write a `manifest.json` file into it.
   - Run `edgeworks-bundle create --image test:1.0 --output <temp-dir>`.
   - Assert stderr contains "already contains" or "exists".
   - Assert exit code is non-zero.

3. **`test_create_invalid_image_ref`** — Verify that an image reference without a tag produces an error.
   - Run `edgeworks-bundle create --image registry/repo --output <temp-dir>`.
   - Assert stderr contains "tag".
   - Assert exit code is non-zero.

4. **`test_create_json_error_output`** — Verify that `--json` mode outputs JSON on error.
   - Run `edgeworks-bundle create --json --image test:1.0 --output <existing-bundle-dir>`.
   - Parse stdout as JSON.
   - Assert `status` field equals `"error"`.
   - Assert `message` field is non-empty.

5. **`test_create_help_shows_flags`** — Verify `create --help` shows `--image`, `--output`, `--notes`, `--target-device`.
   - Run `edgeworks-bundle create --help`.
   - Assert stdout contains each flag name.

**Note:** Full end-to-end tests that actually pull images require skopeo and network access. These tests focus on error paths and CLI argument parsing that can run in any environment. Add a `#[ignore]` test `test_create_e2e_with_skopeo` with a comment explaining it requires skopeo and network access, which can be run manually:

6. **`#[ignore] test_create_e2e_with_skopeo`** — (requires skopeo + network)
   - Pull a small public image (e.g., `docker.io/library/alpine:3.19`).
   - Output to a temp dir.
   - Assert the temp dir contains exactly 3 files: `manifest.json`, `checksums.sha256`, `alpine-3.19.oci.tar`.
   - Parse `manifest.json` and verify fields: `schema_version == "1.0"`, `image.version == "3.19"`, `image.reference` matches input.
   - Run `sha256sum -c checksums.sha256` in the output dir and assert exit code 0.
   - Run with `--json` and parse the JSON output, assert `status == "ok"`.
</description>
</task>

## Verification

