# Plan 05-01: Create command core — skopeo pull, SHA256, bundle output

## Goal

Implement the full `create` subcommand logic in `crates/bundle-cli/src/create.rs`. After this plan, running `edgeworks-bundle create --image <ref> --output <dir>` produces a valid bundle directory containing `manifest.json`, `checksums.sha256`, and `<name>.oci.tar`.

## Context

Phase 4 already created the crate scaffolding at `crates/bundle-cli/` with modules `main.rs`, `create.rs`, `verify.rs`, `inspect.rs`, `manifest.rs`, `error.rs`. The `BundleManifest`, `BundleImage` structs and `BundleError` enum already exist. The `create` subcommand is already wired in `main.rs` via clap but the handler in `create.rs` is a stub. This plan fills in the full implementation.

**Design authority:** `docs/bundle-cli-design.md` sections 2, 3.1, 4, 6.

<task id="05-01-02">
<title>Implement create::run() with full bundle creation pipeline</title>
<description>
Implement the `create` command in `crates/bundle-cli/src/create.rs`. The function signature should accept the parsed CLI args (image ref, output dir, notes, target_device, json flag) and return `Result<(), BundleError>`.

**Implementation steps (match design doc section 3.1 exactly):**

1. **Validate inputs:**
   - Check that `skopeo` is in PATH using `std::process::Command::new("skopeo").arg("--version").output()`. If not found, return `BundleError::SkopeoNotAvailable`.
   - Parse image reference to extract version from tag. The image ref format is `registry/repo:tag`. Extract `tag` as version. If no `:` tag separator exists, return `BundleError::PullFailed("image reference must include a tag (e.g., registry/repo:1.2.0)".into())`.
   - Check if output directory already exists AND contains files. If it contains `manifest.json`, return `BundleError::OutputExists`. If the directory doesn't exist, create it with `std::fs::create_dir_all()`.

2. **Pull image via skopeo:**
   - Construct tarball filename: `edge-os-<version>.oci.tar` (e.g., `edge-os-1.2.0.oci.tar`)
   - Build skopeo command:
     ```
     skopeo copy docker://<image-ref> oci-archive:<output-dir>/<tarball-filename>
     ```
   - Show a spinner progress bar using `indicatif::ProgressBar::new_spinner()` with message `"Pulling image..."` during the skopeo copy.
   - Execute with `std::process::Command`, capture stdout and stderr.
   - If exit code is non-zero, return `BundleError::PullFailed(stderr_string)`.

3. **Compute SHA256 checksum:**
   - Open the tarball file, create `sha2::Sha256` hasher.
   - Read in 8KB chunks, updating the hasher. Show a progress bar using `indicatif::ProgressBar::new(file_size)` with style `"[{bar:40}] {bytes}/{total_bytes} ({eta})"` and message `"Computing checksum..."`.
   - Format digest as `sha256:<hex>`.

4. **Get file size:**
   - Use `std::fs::metadata()` to get `size_bytes`.

5. **Write `checksums.sha256`:**
   - Format: `<hex-digest>  <tarball-filename>\n` (two spaces between hash and filename — GNU coreutils format).
   - Write to `<output-dir>/checksums.sha256`.

6. **Write `manifest.json`:**
   - Construct `BundleManifest` with:
     - `schema_version`: `"1.0"`
     - `created_at`: `chrono::Utc::now()`
     - `created_by`: `format!("edgeworks-bundle v{}", env!("CARGO_PKG_VERSION"))`
     - `image`: `BundleImage { reference, file, digest, size_bytes, version }`
     - `target_device`: from CLI arg, default `"any"`
     - `notes`: from CLI arg, default `""`
   - Serialize with `serde_json::to_string_pretty()`.
   - Write to `<output-dir>/manifest.json`.

7. **Print human-readable summary (non-JSON mode):**
   ```
   Bundle created successfully.
     Directory: <output-dir>
     Image:     <image-ref>
     Size:      <human-readable size>
     Digest:    <digest>
     Files:     3 (manifest.json, checksums.sha256, <tarball-filename>)
   ```
   Use a helper to format bytes as human-readable (e.g., "2.0 GiB").

**Wire into main.rs:**
- In `main.rs`, the `create` subcommand match arm should call `create::run()` with the parsed args. If Phase 4 left a `todo!()` or `unimplemented!()` stub there, replace it.
- On error, print the error message to stderr and exit with code 1.

**Important implementation details:**
- The `checksums.sha256` file must use exactly two spaces between hash and filename (GNU format), so `sha256sum -c` works.
- The digest in `manifest.json` uses `sha256:` prefix format, but `checksums.sha256` uses bare hex.
- Progress bars must be cleared/finished before printing the summary.
</description>
</task>

## must_haves

These are derived backward from the phase goal: "Users can run `edgeworks-bundle create` to produce a valid, verifiable bundle directory from an OCI image reference."

- [ ] `create::run()` function exists and is callable from `main.rs`
- [ ] skopeo is invoked with `skopeo copy docker://<ref> oci-archive:<path>`
- [ ] SHA256 computed via `sha2` crate with streaming reads
- [ ] `checksums.sha256` written in GNU coreutils two-space format
- [ ] `manifest.json` written with all required fields per design doc section 2.2
- [ ] Bundle directory contains exactly 3 files: manifest.json, checksums.sha256, <name>.oci.tar
- [ ] Progress bars shown during pull and checksum computation
- [ ] `BundleError::SkopeoNotAvailable` returned when skopeo missing
- [ ] `BundleError::OutputExists` returned when output dir already has a bundle
- [ ] `BundleError::PullFailed` returned with stderr content on skopeo failure
- [ ] Image tag parsed from reference to derive version and tarball filename
